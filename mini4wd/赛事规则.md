# 迷你四驱车赛事 — 晋级与淘汰规则详解

> 适用:Bmax 组 / 公开组 / 盒车组,3 轨 / 5 轨赛道
> 文档生成日期:2026-05-22

---

## 共通基础概念

### 1. 轮次类型

赛事按"轮次"推进,每轮结束后系统**自动**完成晋级判定 + 下一轮分组。所有轮次按 `round_config` JSON 配置串联:

| 轮次 | 标识 | 含义 |
|---|---|---|
| 第一轮 | `ROUND_1` | 所有报名者参赛 |
| 复活赛 | `REPECHAGE` | 第一轮淘汰者重新争夺机会 |
| 过渡轮 | `ROUND_2` / `ROUND_3` ... | 半决赛前的中间轮(人数太多时使用) |
| 四分之一决赛 | `QUARTER`(由过渡轮承担) | 12~9 人 / 25~10 人时的特殊一轮 |
| 半决赛 | `SEMI` | 固定 `lanes × N` 组,产出决赛入选者 |
| 决赛 | `FINAL` | 固定 1 组,产出冠/亚/季军 |

### 2. 关键参数(round_config 字段)

| 字段 | 含义 |
|---|---|
| `advancePerGroup` | 每组晋级人/队数 |
| `mandatory` | true=必进制(同分必须加赛决出);false=非必进(可 0 晋级) |
| `regroup` | 该轮结束后是否重新分组 |
| `source` | `advanced`(默认,取晋级者)/ `eliminated`(取淘汰者,复活赛专用) |
| `fixedGroups` | 该轮固定分组数(半决赛/决赛/动态倒推用) |
| `advanceUnit` | `INDIVIDUAL` / `TEAM`(组队赛按队晋级) |

### 3. 同分加赛(所有必进制轮)

| 场景 | 处理 |
|---|---|
| 组内同分**且**影响晋级名次 | 强制加赛决出胜负(操作员手动选定 +2 分队员/选手) |
| 全员 0 分 | 同样需要加赛 |
| 同分但不影响晋级(已被取或已被淘汰) | 不需要加赛 |

非必进制(第一轮)允许同分共存,不强制加赛。

# 第一部分:个人赛

## 一、个人赛轮次完整流程

```
ROUND_1 (第一轮)
   ├─ 晋级者 ────────────────────┐
   └─ 淘汰者 → REPECHAGE (复活赛) │
                ├─ 复活晋级者 ────┤
                └─ 复活淘汰者 → 出局
                                ↓
                  合并 R1 晋级 + 复活晋级
                                ↓
              过渡轮(人数 > 半决赛容量时)
                                ↓
                          SEMI (半决赛)
                                ↓
                          FINAL (决赛)
```

各轮可通过 `hasRepechage` / `hasSemiFinal` 开关跳过。

## 二、个人赛晋级判定算法

来源:`TournamentAdvancer.resolveIndividualTargets`

每个选手的成绩有一个 `raceStatus` 字段,据此判定晋级或淘汰:

| `raceStatus` | 中文 | 判定 |
|---|---|---|
| `NORMAL` | 完赛晋级 | **晋级**(参与排名) |
| `FINISHED` | 完赛不晋级 | **淘汰**(本组内已完赛但不再继续) |
| `OFF_TRACK` | 飞车/出轨 | **淘汰** |

> 录分页(`SpectatorView`)非决赛轮显示 3 态:`ADVANCED`(晋级) / `FINISHED`(完赛) / `OFF_TRACK`(飞车),分别映射到上表的 raceStatus。决赛只有 `NORMAL` / `OFF_TRACK` 两态。后端晋级判定逻辑(`TournamentAdvancer`)仅认 `NORMAL` = 晋级,其余一律淘汰。

**晋级人数控制**:
- 每组按用时升序排序(用时短的在前)
- 取前 `advancePerGroup` 名晋级
- 同用时(并列名次)且影响晋级 → 触发同分加赛(仅必进制轮)

## 三、个人赛各轮默认配置

| 轮次 | 字段 | 3 轨默认 | 5 轨默认 |
|---|---|---|---|
| 第一轮 | `r1AdvancePerGroup`(每组晋级数) | **2** | **4** |
| 第一轮 | `r1Mandatory`(必进制) | **false**(允许 0 晋级) | **false** |
| 复活赛 | `repAdvancePerGroup` | 2 | 2 |
| 复活赛 | `repMinAdvancePerGroup`(最小晋级) | 1 | 1 |
| 复活赛 | 隐式 mandatory | **true** | **true** |
| 过渡轮 | `transitionAdvancePerGroup` | 2 | 2 |
| 过渡轮 | `transitionMinAdvancePerGroup` | 1 | 1 |
| 半决赛 | `semiAdvancePerGroup` | **1** | **1** |
| 半决赛 | `semiFixedGroupsMultiplier`(固定组数倍数) | 1(= 3 组) | 1(= 5 组) |
| 决赛 | `advancePerGroup` | = 轨数(3) | = 轨数(5) |
| 决赛 | `fixedGroups` | **1** | **1** |

## 四、个人赛第一轮(ROUND_1)

- 全部报名者参赛,自动随机打乱后分组
- 每组人数 ≤ 赛道轨数(3 或 5)
- 录分完成后,每组按用时取**前 2 名**(3 轨)/ **前 4 名**(5 轨)晋级
- **非必进制**:若某组完赛人数不足 / 表现差,允许该组少于默认数量晋级
- 未晋级者全部进入复活赛

## 五、个人赛复活赛(REPECHAGE)

### 5.1 触发条件
- `hasRepechage: true`(默认开启)
- 参赛者来源:第一轮被淘汰的所有人(`source: eliminated`)

### 5.2 复活赛分组数动态倒推

为了让"第一轮晋级 + 复活赛晋级"对齐**轨道数整数倍**(避免出现"零头组"),系统按以下算法:

```
r1Advance     = 第一轮总人数 - 淘汰池
defaultPool   = r1Advance + ⌈r1Eliminated / lanes⌉   ← 朴素假设每组取 1 人
alignedPool   = ⌈defaultPool / lanes⌉ × lanes        ← 向上对齐到 lanes 整数倍
若 alignedPool > 总人数 → 向下对齐 = (总人数 / lanes) × lanes
repNeed       = alignedPool - r1Advance               ← 复活赛需要晋级的人数
fixedGroups   = repNeed                                ← 复活赛分组数(每组取 1 人)
```

### 5.3 个人赛复活赛示例(18 人 3 轨)

| 步骤 | 计算 | 结果 |
|---|---|---|
| 第一轮 | 18 人 6 组(3+3+3+3+3+3) | r1Advance=7 / r1Eliminated=11 |
| 默认分组 | ⌈11/3⌉ | 4 组(原算法,有零头) |
| **倒推算法** | defaultPool=7+4=11 → alignedPool=12 | **5 组**(每组 2~3 人) |
| 复活赛晋级 | 5 组 × 1 人 | 5 人 |
| **第二轮总人数** | 7 + 5 | **12 人**(4 组 × 3 人 ✓) |

### 5.4 复活赛规则
- **必进制**(隐式 mandatory=true):同分必加赛
- 每组按用时取 1 人晋级
- 复活赛未晋级者彻底出局

## 六、个人赛跨轮合并(复活赛后)

第二轮开始时,需要把**第一轮晋级者**和**复活赛晋级者**合并到同一池子,再重新打散分组。

实现方式:
- `regroupNext` 接受 `prevAdvancedRound = 'ROUND_1'` 参数
- 后端从 `ROUND_1` 取所有 NORMAL 选手 + 复活赛 NORMAL 选手 → 合并
- 重新随机打乱分组到下一轮

## 七、个人赛过渡轮 / 四分之一决赛

前端 SpectatorView 在跳轮时按如下规则决定下一轮:

记 `advancingCount` = 本轮晋级人数,`lanes` = 轨道数:
- `lanes × 2 = semiMin`(半决赛最少人数)
- `lanes × lanes = semiMax`(半决赛最多人数)

| 条件 | 下一轮 | fixedGroups |
|---|---|---|
| `advancingCount ≤ lanes`(≤3 / ≤5) | **直接 FINAL** | 1 |
| `advancingCount ∈ [semiMin, semiMax]`(6~9 / 10~25) | **SEMI 半决赛** | `lanes`(3 或 5) |
| `advancingCount ≥ semiMin × 2`(≥12 / ≥20) | **四分之一决赛** | `max(⌈人数/lanes⌉, semiMin)` |
| 其他 | round_config 中下一个过渡轮 | — |

注:`ROUND_1` 和 `REPECHAGE` 后**不允许跳轮**,必须按 round_config 顺序走。

## 八、个人赛半决赛(SEMI)

- **固定分组数** = `lanes × semiFixedGroupsMultiplier`(默认 1,即 3 组 / 5 组)
- 每组按用时取 **1 人**进决赛
- **必进制**:同分必加赛
- 3 轨:9 人 → 3 组 × 3 人 → 取 3 人进决赛
- 5 轨:15~25 人 → 5 组 → 取 5 人进决赛

## 九、个人赛决赛(FINAL)

- **固定 1 组**,人数 = 决赛进入人数(3 人 / 5 人)
- 按用时排名:1 名冠军 / 2 名亚军 / 3 名季军
- 系统自动生成对应荣誉(`CHAMPION` / `RUNNER_UP` / `THIRD_PLACE`)

# 第二部分:组队赛

## 一、组队赛轮次完整流程

```
ROUND_1 (第一轮 — 每组多支队伍, 每队按顺位赛多场)
   ├─ 晋级队 ────────────────────┐
   └─ 淘汰队 → REPECHAGE (复活赛) │
                ├─ 复活晋级队 ───┤
                └─ 复活淘汰队 → 出局
                                ↓
                  合并 R1 晋级 + 复活晋级
                                ↓
              过渡轮(队伍数 > 半决赛容量时)
                                ↓
                          SEMI (半决赛)
                                ↓
                          FINAL (决赛 — 各队队员按顺位定名次)
```

## 二、组队赛比赛规则

### 2.1 顺位赛制(每组多场)
- 一组多支队伍(默认 `teamsPerGroup = lanes`,即每组 3 队 / 5 队)
- 每队 N 名队员,按顺位赛 N 场:
  - **第 1 场**:各队 1 号位车手对决
  - **第 2 场**:各队 2 号位车手对决
  - **第 3 场**:各队 3 号位车手对决 ...
- 每场单人按完赛名次得分

### 2.2 单场得分规则

| 完赛名次 | 3 轨得分 | 5 轨得分 |
|---|---|---|
| 第 1 | 9 | 9 |
| 第 2 | 7 | 7 |
| 第 3 | 5 | 5 |
| 第 4 | — | 3 |
| 第 5 | — | 1 |
| 未完赛 / 弃权 / 故障 / 罚时未达标 | **0** | **0** |

由 `system_config.team_scoring_3v` / `team_scoring_5v` 配置,默认 "9,7,5,3,1,0"。

### 2.3 队伍总分
- 队伍总分 = 该队所有队员所有顺位场次得分之和
- 例:5 队 × 3 人,3 场比赛后队伍总分最高 27 分(全场 9 分)
- 总分相同 → 同分平局(必进轮触发加赛)

## 三、组队赛晋级判定算法

来源:`TournamentAdvancer.resolveTeamTargets`

- 遍历所有成绩,按 `raceTeamId` 分组
- **同一队伍只要至少一名队员是 NORMAL** → 该队整体算晋级
- 全队所有顺位都非 NORMAL → 整队淘汰
- **晋级取队**:按 `advancePerGroup`(每组取 N 队)
- **同分排名**:并列时需必进轮加赛(`mandatory: true`)

### 3.1 跨组总分截取(附加赛 / `totalAdvance`)
- 多组合并后,如果晋级队数超过下一轮容量,按**跨组总分降序**取前 N 队
- 用于半决赛/决赛之前精确控制队伍数

## 四、组队赛各轮默认配置

| 轮次 | 字段 | 3 轨/5 轨默认 |
|---|---|---|
| 第一轮 | `r1AdvancePerGroup`(每组晋级队数) | **1**(最多 1 队晋级,允许 0) |
| 第一轮 | `r1Mandatory` | **false** |
| 复活赛 | `repAdvancePerGroup` | **1** |
| 复活赛 | `repMinAdvancePerGroup` | null |
| 复活赛 | 隐式 mandatory | **true** |
| 过渡轮 | `transitionAdvancePerGroup` | **1** |
| 过渡轮 | `transitionMinAdvancePerGroup` | null |
| 半决赛 | `semiAdvancePerGroup` | **1** |
| 半决赛 | `semiFixedGroupsMultiplier` | 1 |

## 五、组队赛第一轮(ROUND_1)

- 所有报名队伍参赛,自动随机打乱后分组
- 每组队伍数 = `teamsPerGroup`(默认 = 轨数)
- 每组按顺位赛多场,得出每队总分
- 按总分降序取**前 1 队**(默认)晋级 — 即"每组最多 1 队晋级"
- **非必进制**:若全组总分都很低(或全 0),允许该组 0 晋级
- 未晋级者全部进入复活赛

## 六、组队赛复活赛(REPECHAGE)

### 6.1 触发条件
- `hasRepechage: true`(默认开启)
- 参赛者:第一轮被淘汰的所有队伍

### 6.2 复活赛分组数动态倒推(组队赛专属)

来源:`calculateRepechageFixedGroups`

```
r1Actual          = 第一轮晋级队数
target            = semiMin (= lanes × 2)
按过渡轮数 transitionCount 倍增: target = target × 2 × transitionCount
继续倍增直到 target > r1Actual
repNeed           = max(1, target - r1Actual)
repNeed           = min(repNeed, 淘汰池 / 2)          ← 限制不超过淘汰池一半
minByTrack        = ⌈淘汰池 / lanes⌉
repNeed           = max(repNeed, minByTrack)          ← 保证分组人数合理
fixedGroups       = repNeed
```

### 6.3 组队赛复活赛示例(12 队 3 轨,第一轮 4 组)

| 步骤 | 计算 | 结果 |
|---|---|---|
| 第一轮 | 12 队 4 组(每组 3 队) | r1Advance=4 / r1Eliminated=8 |
| target 计算 | semiMin=6 → 倍增到 > 4 | target=6 |
| repNeed(初) | 6 − 4 | 2 |
| min(repNeed, 4) | 限制不超淘汰池一半 | 2 |
| minByTrack | ⌈8/3⌉ | 3 |
| 最终 repNeed | max(2, 3) | **3 组** |
| 复活赛晋级 | 3 组 × 1 队 | 3 队 |
| **第二轮总队数** | 4 + 3 | **7 队**(进入下一轮) |

### 6.4 复活赛规则
- **必进制**:同分必加赛
- 每组按总分取 1 队晋级
- 复活赛未晋级者出局

## 七、组队赛跨轮合并

跟个人赛逻辑一致:
- `regroupNext` 加 `prevAdvancedRound = 'ROUND_1'`
- 后端合并 R1 晋级队 + 复活晋级队
- 重新打散分组

## 八、组队赛过渡轮 / 四分之一决赛

前端跳轮逻辑:
- 与个人赛**完全相同**,只是单位从"人"变成"队"
- `advancingCount` 含义为晋级队伍数

## 九、组队赛半决赛(SEMI)

- **固定分组数** = `lanes × semiFixedGroupsMultiplier`(默认 = 轨数)
- 每组按总分取 **1 队**进决赛
- **必进制**:同分必加赛
- 3 轨:3 组 × 3 队 = 9 队 → 取 3 队进决赛
- 5 轨:5 组 × 5 队 = 25 队 → 取 5 队进决赛

## 十、组队赛决赛(FINAL)

- **固定 1 组**,所有决赛队伍同台
- 按总分排名:第 1 = 冠军队、第 2 = 亚军队、第 3 = 季军队
- **每个队员各获得一条荣誉**(`race_team_id` 关联队伍),保证公平
- 多队同分最高时,本组顶部 UI 自动高亮多队(同时高亮)

# 第三部分:同分加赛规则(共通)

## 1. 触发条件

只在**必进制**(`mandatory: true`)轮次触发:
- 复活赛、过渡轮、半决赛、决赛默认必进
- 第一轮默认非必进(可同分共存)

## 2. 加赛场景

| 场景 | 行为 |
|---|---|
| 个人赛同用时 + 影响晋级 | 加赛决出 |
| 组队赛同总分 + 影响晋级 | 加赛决出 |
| 全员 0 分 | 加赛决出 |
| 已经全员晋级或全员淘汰(同分不影响名次) | 不加赛 |

## 3. 加赛操作流程

1. admin 在录分页看到提示:**"⚠ 第 N 组存在同分平局,请先加赛决出胜负"**
2. 操作员手动选定加分队员 / 选手
3. 系统给该选手/队员 **+2 分**
4. 重新计算总分 / 排名
5. 直到能明确决出名次

# 第六部分:术语速查

| 术语 | 含义 |
|---|---|
| `advancePerGroup` | 每组晋级人/队数 |
| `advanceTop` | 总晋级数(跨组总分截取,附加赛专用) |
| `fixedGroups` | 该轮固定分组数(半决赛、决赛、复活赛用) |
| `mandatory` | 必进制(true 必须晋级 advancePerGroup;false 允许 0~N) |
| `source` | `advanced` 取晋级者 / `eliminated` 取淘汰者(复活赛专用) |
| `prevAdvancedRound` | 前一轮(用于跨轮合并 R1 晋级者) |
| `regroup` | 该轮结束后是否重新分组(true)还是延续上轮分组(false) |
| `r1*` | 第一轮配置前缀(r1AdvancePerGroup 等) |
| `rep*` | 复活赛配置前缀 |
| `semi*` | 半决赛配置前缀 |
| `transition*` | 过渡轮配置前缀 |
| `lanes` | 赛道轨数(3 或 5) |
| `teamsPerGroup` | 组队赛每组队伍数(= lanes) |
